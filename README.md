# AIエージェント スキルリポジトリ

このリポジトリは、AIエージェントが特定のタスクを遂行するための「スキル」を管理する場所です。各スキルは、特定のペルソナ、文体、知識、またはタスク実行手順を定義しており、AIエージェントがより高品質で専門的なアウトプットを生成するために利用されます。

## 🚀 スキルの使い方

各スキルは独立したディレクトリに格納されており、そのスキルをAIエージェントに適用するための詳細な指示書（`SKILL.md`）が含まれています。

1.  **スキルディレクトリの確認**: `skills/` ディレクトリ内で、利用したいスキルのディレクトリ（例: `skills/investor_f/`）を探します。
2.  **`SKILL.md` の読み込み**: 各スキルディレクトリ内の `SKILL.md` ファイルを開き、内容をAIエージェントのシステムプロンプトや指示書として読み込ませます。
3.  **適用**: `SKILL.md` に記載されている指示に従い、AIエージェントにタスクを実行させます。

## 📂 リポジトリ構造

```
github_skills/
├── README.md             # このファイル。リポジトリ全体の概要と使い方
├── skills/               # 各スキルを格納するディレクトリ
│   └── [skill_name]/     # スキルごとのディレクトリ（例: investor_f）
│       ├── SKILL.md      # スキルの詳細な指示書（AIエージェント用）
│       └── [関連ファイル]  # スキルに関連するスクリプト、データ、設定ファイルなど
└── templates/            # 新しいスキルを作成するためのテンプレート
    └── SKILL_TEMPLATE.md # 新規スキル作成用のテンプレートファイル
```

## ✨ 新しいスキルの追加方法

新しいスキルを追加する際は、`templates/SKILL_TEMPLATE.md` をコピーし、以下の手順で作成してください。

1.  `skills/` ディレクトリ内に、新しいスキル名（例: `data_analyst`）でディレクトリを作成します。
    `mkdir -p skills/[新しいスキル名]`
2.  `templates/SKILL_TEMPLATE.md` を新しいスキルディレクトリにコピーし、`SKILL.md` にリネームします。
    `cp templates/SKILL_TEMPLATE.md skills/[新しいスキル名]/SKILL.md`
3.  新しい `SKILL.md` ファイルを編集し、以下の項目を埋めます。
    *   **スキル名**: スキルの簡潔な名称。
    *   **概要**: スキルがどのようなタスクを目的としているか。
    *   **ペルソナ**: AIエージェントがどのような役割を演じるか（一人称、口調、専門性など）。
    *   **禁止表現**: そのペルソナが避けるべき言葉や表現。
    *   **推奨表現**: そのペルソナが積極的に使うべき言葉や表現。
    *   **タスク実行手順**: スキルを適用した際の具体的なタスク実行フロー。
    *   **関連ファイル**: スキルに関連するスクリプトや設定ファイルなどがあれば記載。

この手順に従うことで、一貫性のある高品質なスキルを効率的に追加・管理できます。
